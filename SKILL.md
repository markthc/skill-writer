# Skill Writer

交互式 Skill 创作助手 - 通过多轮对话确认需求后再生成 Skill。

## 核心原则

**慢思考，快执行**。不急于产出，确保 Skill 质量。

---

## 阶段 1：需求澄清

### 1.1 场景探询

```
请描述一个具体的使用场景：
- 什么时候会用到这个 Skill？
- 当时你在做什么？
- 期望 AI 帮你解决什么问题？
```

### 1.2 两个变量确认

| 变量 | 问题 |
|-----|------|
| **A（输入）** | 用户提供什么？格式？必填/可选？ |
| **B（输出）** | AI 产出什么？格式？验收标准？ |

### 1.3 边界定义

```
管什么（必须处理）：___
不管什么（明确排除）：___
```

---

## 阶段 2：骨架选择

| 类型 | 适用场景 |
|-----|---------|
| 单步骤型 | 一次性转换（翻译、格式化） |
| 顺序型 | 多步骤按顺序执行 |
| 分支型 | 根据条件选择不同路径 |
| 迭代型 | 循环改进直到满足条件 |
| 递归型 | 处理嵌套结构 |

向用户确认骨架类型及原因。

---

## 阶段 3：示例收集

**关键步骤**：索要 1-2 个具体输入/输出示例。

---

## 阶段 4：生成草稿

### Skill 模板

```markdown
---
name: [skill-name]
description: Use when [触发条件，不写工作流程]
---

# [Skill 名称]

## 概述
核心原则 1-2 句

## 何时使用
- 触发条件
- 不适用场景

## 输入（变量 A）
| 字段 | 类型 | 必填 | 说明 |

## 输出（变量 B）
| 字段 | 类型 | 说明 |

## 边界
- 管什么：...
- 不管什么：...

## 执行步骤
### [步骤名]
[具体指令]

## 示例
**输入**：...
**输出**：...

## 常见错误
| 错误 | 修正 |
|-----|------|
```

---

## 阶段 5：验证迭代

生成后询问用户：
1. 输入/输出是否符合预期？
2. 步骤是否清晰可执行？
3. 边界是否准确？
4. 示例是否覆盖典型场景？

---

## CSO（Claude 搜索优化）

**关键发现**：Description 如果总结了工作流程，Claude 会跳过 skill 正文！

### Description 写法

```yaml
# ❌ 错误：总结了工作流程
description: 交互式创作 - 通过多轮对话确认需求后生成 Skill

# ✅ 正确：只写触发条件
description: Use when user says "创建 skill" or wants to create new skills
```

**规则**：
- 以 "Use when..." 开头
- 只描述触发条件/症状
- 不描述工作流程
- 第三人称
- < 500 字符

---

## 对话风格

**要做的**：
- 每次只问 1-2 个问题
- 用具体例子帮助理解
- 复述需求确认理解

**不要做的**：
- 一次性问太多问题
- 假设用户需求
- 跳过示例收集

---

## 快速模式

用户说"快速生成"或提供完整需求文档时：

```
检测到快速模式，确认理解：
- [理解 1]
- [理解 2]
回复"确认"继续，或提供修正。
```

---

## 输出

```
~/.claude/skills/[skill-name]/SKILL.md
```

命名规则：小写字母 + 连字符，简短描述性强
